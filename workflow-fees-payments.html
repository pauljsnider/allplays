Manage Team Fees and Payments

Set up fee records, track payments, and let parents pay online via Stripe or record offline payments.

4 min read Updated from live product pages

On this page

Who can use this

Parent Coach Admin

Overview

Team fees can be created by admins, paid by parents via Stripe Checkout, or tracked as offline payments. Admins can record balance adjustments, configure installment schedules, and cancel fees at any time. Parents see a full invoice view for each fee — including total amount, amount paid, remaining balance, individual line items, and the installment schedule if one is set up. Payment metadata from Stripe (session ID, payment intent, paid amount, and receipt) is stored directly on the fee record so admins always have a complete audit trail. Failed or expired Stripe sessions never mark a fee as paid.

Who Is This For

Admin: Create and configure fee records, set recipient lists, define installment schedules, generate Stripe Checkout links, record offline payments, apply manual adjustments, cancel fees, and monitor per-recipient payment status across the team.

Coach: Coaches with admin access can create and manage fees. Coaches without admin access can view fee status for their team.

Parent: View the invoice for each fee assigned to your player, see the full breakdown of amounts and due dates, and pay online when a Stripe Checkout link is available.

Prerequisites

  • Signed in to ALL PLAYS with your account
  • For online payment, a Stripe Checkout link must be generated by the admin first — parents cannot initiate online payment without it
  • For parent invoice view, the parent account must be linked to the team (listed in parentTeamIds)
  • Admin role required to create fees, record adjustments, or cancel fee records

Choose Your Path

  • Set up a new fee — admin creates a fee record with the recipient list, total amount, and optional installment schedule
  • Record an offline payment — admin uses a ledger adjustment to note cash, check, or other non-Stripe payments
  • Parent wants to pay online — parent opens the fee detail from the parent dashboard, then selects Pay to go to Stripe Checkout
  • Adjust or cancel a fee — admin uses manual adjustment controls to change the balance or cancel the fee entirely

Step-by-Step Workflow

  1. Admin: Create the fee record. Go to team management and open the Fees section. Click New Fee, enter the fee name, select recipients from the roster, set the total amount, and optionally configure an installment schedule with due dates and per-installment amounts.
  2. Admin: Generate a Stripe Checkout link. From the fee record, click Generate Payment Link. This creates a Stripe Checkout session and stores the URL on the fee. Copy the link or leave it for parents to access through their dashboard.
  3. Parent: Open the fee in the Parent Dashboard. Navigate to Parent Dashboard → Fees and expand the fee to see the full invoice — total amount, amount paid, remaining balance, individual line items, and installment schedule with due dates.
  4. Parent: Pay online. The Pay button appears only when a Stripe Checkout URL exists and the fee is unpaid or partially paid. Clicking Pay redirects to Stripe Checkout to complete the payment securely.
  5. Stripe processes the payment. On success, payment metadata is written to the fee record: Stripe session ID, payment intent, amount paid, and a receipt URL. The fee status updates to paid or partial based on the amount received. Failed or expired Stripe sessions never change the fee status — the fee remains unpaid until a successful payment is confirmed.
  6. Admin: Record an offline payment. Open the fee record, click Add Adjustment, set the type to Offline Payment, enter the amount, and add a reason (e.g., "Cash received at practice 2026-04-10"). The reason field is required for all adjustments.
  7. Admin: Apply a manual balance adjustment or cancel. Use Add Adjustment for positive or negative balance changes — for example, applying a scholarship discount (negative) or adding a late fee (positive). To cancel the fee entirely, use the Cancel Fee option; canceled fees are removed from the parent invoice view.
  8. Admin: Review per-recipient payment status. The fee record shows each recipient's current status: Paid, Unpaid, Partial, or Canceled, along with the remaining balance for each. Use this view to follow up with families who have outstanding balances.

Common Questions

Can a parent pay in installments?

Yes, if the admin configured an installment schedule when creating the fee. Parents see each installment with its amount and due date in the invoice view. Each installment can be paid separately when the admin generates a corresponding Stripe link for it.

What if the Stripe session expires before the parent pays?

Stripe Checkout sessions expire after a set time. If the session expires, the fee status stays unpaid. The admin needs to generate a new Stripe Checkout link so the parent can try again.

Can an admin mark a fee as paid without going through Stripe?

Yes. Use the Add Adjustment option with type Offline Payment and enter the full amount received. This records the payment in the ledger and updates the fee status without involving Stripe.

Who can see fee records?

Admins see all fee records and recipient details. Parents see only their own player's fee invoice — they cannot see other families' fee information or payment history.

Recovery and Troubleshooting

Pay button is not showing for a parent: Confirm that a Stripe Checkout URL has been generated by the admin for this fee. Also confirm the fee is not already in a paid or canceled state — the Pay button is hidden for completed or canceled fees.

Stripe session failed or expired: The fee status remains unpaid. No partial credit is applied for failed sessions. The admin should generate a new Checkout link and share it with the parent.

Adjustment is not saving: The reason field is required for all ledger adjustments. Make sure the reason field is filled in before submitting. The form will not save with an empty reason.

Parent cannot see their fee: The parent account must be linked to the team. Check that the parent's account appears in the team's parent list. If not, add them via team settings and ask them to refresh the parent dashboard.
