Calendar & Schedule

Schedule Private Team Calendar Feed (Token-Authorized ICS)

Each team member gets a personal, token-authorized HTTPS ICS subscription URL that includes games, practices, RSVP status, and officiating assignments. The SHA-256 token is tied to the member's identity and can be revoked. Subscribe via Apple Calendar, Google Calendar, or any ICS-compatible app using the new Sync Calendar dialog.

→ Schedule: Private Calendar Feed
Schedule Public Fan Calendar Feed

Public teams and teams with shareable games expose a read-only games-only ICS feed for fans. Excludes practices, RSVPs, and assignments. Accessible without a login. URL available from the schedule editor for sharing with league pages or club websites.

→ Schedule: Public Fan Feed

Platform admins generate a full season slot grid across all teams. Input: team selection, season date range, venue availability windows, blackout dates, and game duration. Generator produces a conflict-free draft, which admins review, edit, and save locally before publishing. Published slots write to each team's schedule; mirror write failures trigger a rollback to keep all teams consistent.

AI Features

AI AI Coach Focus (Pre-Game)

Runs from the Game Day pre-game panel. Reviews previous game event data and current player availability (Going/Maybe/Out) to produce a short tactical emphasis for today's game — typically one to three sentences with a specific positional or matchup focus. Suggested emphasis, not a mandatory instruction.

→ Game Day: AI Actions
AI Analyze Game (Post-Game AI)

Runs from the Wrap-Up panel after the final whistle. Sends complete game events, substitutions, coach logs, and the Wrap-Up final score to the AI backend. Returns a structured analysis (what went well, areas to improve, key player callouts) plus a practice feed linked to drill recommendations in drills.html. Uses the Wrap-Up score — not the in-game running score — for all analysis context.

→ Game Day: AI Actions
AI Generate Game Summary (Editable & Re-saveable)

Writes a natural-language match summary paragraph using the Wrap-Up score, opponent name, and key events. Output appears in an editable field. Save Summary can be triggered any number of times; the game page always shows the most recently saved version. Coaches can generate a draft, save it, revise manually later, and save again without limit.

→ Game Day: AI Actions

Live Tracker Improvements

Game Tracking Clock State Persistence Across Reloads

The Live Tracker now restores elapsed time correctly on page reload. Previously, reloading while the clock was running could lose accumulated time. Clock state (running/stopped, elapsed time) is persisted to Firestore on every tick and restored from the game record when the page initializes.

→ Live Tracker: Clock
Game Tracking Substitution Persistence (liveLineup)

Every time a substitution is applied, the current lineup is written to game.liveLineup in Firestore. On reload, the tracker restores from liveLineup rather than the original pre-game lineup, so sub changes are never lost. Multi-player substitutions are queued and applied atomically.

→ Live Tracker: Substitutions
Game Tracking Foul Tracking with FOULED OUT Alerts

Per-player foul counters display on each player button. When a player reaches the foul limit (5 fouls for basketball), their button shows a FOULED OUT! label and is visually flagged for immediate recognition. Opponent fouls are tracked separately and persist across page reloads — previously opponent foul counts were lost on reload.

→ Live Tracker: Fouls
Game Tracking Zero-Stat Participation Markers

Players who participate in a game but record zero countable stats are now explicitly marked as participants in the aggregated stat record. This prevents them from being omitted from the game report and ensures their playing time is correctly credited when the game is finished.

→ Live Tracker: Stats
Game Tracking Stat Undo Rebroadcasts Corrected Aggregates

When a stat is undone in the Live Tracker, the corrected aggregated stats are now re-broadcast to Game Day Command Center and the live watch view. Previously, the undo would update the internal tracker state but the broadcast totals shown to other users would remain stale until the next stat event was recorded.

→ Live Tracker workflow
Game Tracking Playing Time Saved Correctly on Finish

The game finish flow now correctly computes and saves per-player playing time using the full substitution log from liveLineup history. Previously, players who were subbed back in after a reload had their post-reload time omitted from the final playing time calculation.

→ Live Tracker: Finishing

Game Day Command Center

Game Tracking Auto-Populate Lineup from RSVP Pool

When the Game Day lineup is empty, opening the lineup panel now automatically fills positions with players from the Going RSVP pool. The auto-fill uses the team's configured formation. Coaches review the auto-filled lineup and can drag-and-drop to adjust before saving. Save Lineup is still required — auto-fill does not save automatically.

Coaches and admins can start a direct conversation thread with a specific parent, player, or staff member — separate from the main team chat channel. Conversation visibility is scoped to participants only. Useful for private scheduling discussions, sensitive feedback, or one-on-one check-ins.

Practice sessions that are not linked to a team game are now visible on the parent dashboard. Previously, standalone practices were omitted from the parent view. Previously, the guard was applied after initial data fetch, creating a brief window where config data was visible to unauthorized users.

→ Team Setup workflow
Platform Athlete Profile Name Prefill

When an athlete profile page is opened with teamId and playerId URL parameters pointing to an existing roster entry, the player's name is automatically prefilled in the profile form. Only applies on new profile creation — does not overwrite an existing profile's name field.
